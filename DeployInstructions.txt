GitHub Branch Protection (main)

Open your repository on GitHub.
Go to Settings → Branches.
Under Branch protection rules, click Add rule.
Branch name pattern: main.
Turn on:
Require a pull request before merging
Require approvals (set at least 1)
Require status checks to pass before merging
Require branches to be up to date before merging
Restrict who can push to matching branches
Do not allow bypassing the above settings (include admins)
Save changes.
If your repo uses Rulesets instead:

Settings → Rules → Rulesets → New branch ruleset
Target branch: main
Enable the same controls above
Save and enable ruleset
Netlify Production Branch

Open Netlify for this site.
Site configuration → Build & deploy.
Continuous Deployment → Branches.
Set Production branch to main.
Keep Deploy Previews enabled for PRs.
This gives you:

Feature branches and PRs for safe testing
Main as the only production website deploy trigger

Important Ordering Note
For changes involving schema/functions, keep changes backward-compatible and promote backend first, then merge to main for frontend deploy. That avoids the site deploying code that expects backend changes not yet live.
